## [How AI agent memory works.](https://memory.cobanov.dev/)


下面是对文章 **《How AI agent memory works》** 的介绍摘要。这篇文章用交互式示例解释了：AI Agent 的“记忆”并不是模型本身真的记住了什么，而是围绕模型的系统在每次调用时，如何选择、保存、检索和治理上下文。

### 概览

这篇文章介绍了 AI Agent 记忆的基本原理：语言模型本身是无状态的，而 Agent 的记忆来自外部编排系统对上下文、长期信息、语义检索和记忆生命周期的管理。

### 关键要点：

- **LLM 本身没有长期记忆**：模型每次调用只看到当前输入；所谓“记得之前的对话”，其实是系统把历史上下文重新塞进提示词里。

- **上下文窗口是最简单的记忆方式**：把完整聊天记录持续放入 prompt 中可以实现短期记忆，但会受到上下文长度限制；一旦超限，就必须丢弃、总结或检索部分信息。

- **Agent 通常有两类记忆**：
  - **工作记忆**：当前对话、近期消息、工具调用等，直接放在上下文中。
  - **长期记忆**：被系统认为值得保存的信息，会被持久化，并在需要时检索回来。

- **记忆不是简单存储，而是生命周期管理**：真实系统要处理写入、更新、过期、冲突、遗忘和隐私过滤。例如用户搬家后，系统不应简单覆盖旧地址，而应记录“旧地址已失效、新地址将生效”。

- **语义搜索是长期记忆检索的核心机制**：文章解释了 embeddings 如何把文本转成向量，让系统可以根据“意思相近”而不是关键词匹配来找回相关记忆。

- **不同类型的记忆适合不同用途**：文章借鉴认知科学，将 Agent 记忆划分为多种类型，例如事件记忆、语义记忆等，每种记忆有不同的写入规则、结构和检索方式。

- **好的 Agent 记忆需要治理**：糟糕的系统可能“什么都忘”，更危险的是“记错、记多、记下敏感信息”。因此记忆系统必须具备冲突处理、隐私保护和可审计性。

## [What Is Code?](https://martinfowler.com/articles/what-is-code.html)

《What Is Code?》指出，在 LLM 让代码生成变得廉价的时代，代码真正的价值不只是“给机器执行的指令”，而是承载业务领域概念、共享词汇、抽象边界和系统设计判断的模型。
* 代码有两个层面：一是机器可执行的指令，二是人类理解问题领域的概念模型；前者正在被 AI 商品化，后者仍是开发者的核心价值。

* 写代码本质上是在建模：变量名、类型、接口、方法边界和类层级，都是开发者对业务概念和系统关系的表达。

* 软件开发是一种翻译活动：开发者需要把业务领域词汇翻译成技术领域词汇，再用编程语言构建新的系统词汇。

## [From Vibe Coding to Spec-Driven Development](https://towardsdatascience.com/from-vibe-coding-to-spec-driven-development/)

文章认为，AI 编程正在从早期的 Vibe Coding，也就是“随口描述需求、让 AI 直接写代码、再不断试错”，转向更成熟的 Spec-Driven Development，规格驱动开发。
后者强调先把产品目标、技术栈、路线图、功能需求和验收标准写成可持续维护的规格文档，再让 AI Agent 按照这些规格实现代码，并由人类进行监督和审查。
